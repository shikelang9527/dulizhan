# MewMew Co — 宠物独立站（成品）

按《宠物独立站 2026 最新可执行配置文档》v2.1 落地的**可运行独立站产品**：
**双语前台（英文默认 / 一键切换中文）** + 中文管理后台，完整覆盖 建站 → 支付 → 退款 → 物流 → 邮件 → 数据 → 风控 → 验收 闭环。

## 前台双语说明

- 导航栏右上角 **中文 / EN** 按钮一键切换，选择存 Cookie（一年），也可用 `?lang=zh` / `?lang=en` 直达
- 切换范围：全部 UI 文案、10 个种子产品的完整文案（名称/卖点/利益点/规格/FAQ/尺码表）、6 个政策页、FAQ 页、购物车/结账/订单查询/确认页、错误提示
- 后台产品编辑内有"中文文案（可选）"区块——不填自动回落英文，填了中文站就显示中文
- 顾客邮件保持英文（面向美国市场的交易凭证）；如需双语邮件可后续扩展

## 快速开始

```bash
cd 宠物独立站
npm install        # 仅需一次（express + nodemailer）
npm start          # 启动
```

- 前台：http://localhost:4180
- 后台：http://localhost:4180/admin （**首次打开会引导设置管理员密码**）
- 数据存在 `data/` 目录（JSON 文件），备份 = 拷贝该目录

> 端口可用环境变量改：`PORT=8080 npm start`

## 公网访问（让任何人打开）

**方式一：一键临时公网（已配好，双击即用）**

双击项目里的 **`启动网站.bat`** → 弹出两个窗口 → "MewMew-公网隧道"窗口里 `https://xxx.trycloudflare.com` 就是任何人可打开的公网地址（含 HTTPS）。

原理：Cloudflare Tunnel 免费快速隧道（`tools/cloudflared.exe`），无需注册、无需路由器设置。

局限（重要）：
- 你的电脑必须开机且两个窗口不能关，关了网站就下线
- 公网地址**每次重启都会随机变化**，不适合印在广告/名片上
- 无正常运行时间保证，仅适合演示、给朋友/客户看、手机真机测试

**方式二：正式部署（7×24 在线 + 固定域名，推荐正式运营用）**

任选一个能跑 Node.js 的平台，把本目录传上去：
1. VPS（阿里云/腾讯云海外节点、Vultr、RackNerd 等，$5-10/月）：装 Node 20+ → `npm install` → `npm start`，用 pm2 守护 + Caddy 反代自动 HTTPS
2. PaaS（Railway / Render / Fly.io）：连 GitHub 仓库或直接上传，启动命令 `npm start`，注意把 `data/` 目录挂持久卷（订单数据在里面）
3. 部署后：买域名 → 解析到服务器 → 后台"设置→店铺信息"填入域名（邮件链接和支付回调会用它）

管理后台密码在本地文件 `管理员密码.txt`（已随机生成锁定，可在后台"设置"里修改）。

## 从本地到真实收款的上线路径

1. **确认主体与收款**（文档 §2，只能由你完成）：
   香港公司 / 美国 LLC / 第三方跨境收款 → 开通 Stripe 或 PayPal Business。
   ⚠️ 中国大陆个人身份不能直接开 Stripe，别信"直接 Stripe→Wise→国内卡"的旧教程。
2. **部署**：任意能跑 Node.js 的主机（VPS / Railway / Render 等）：上传本目录 → `npm install` → `npm start`（建议配 pm2 / systemd 守护 + Caddy/Nginx 反代 HTTPS）。
3. **域名**：解析到服务器 → 后台"设置→店铺信息"填入域名（邮件里的链接会用它）。
4. **品牌邮箱**：如 support@你的域名（Zoho Mail 免费版即可）→ 后台"设置→SMTP"填入 → 客服邮箱同步更新。
5. **支付**：后台"设置→支付"切到 Stripe/PayPal 填密钥 → 按文档做 **$50–100 真实收款→退款→提现** 全链路测试。
6. **替换产品**：演示的 10 个产品是带完整文案的示例（插画非实拍）。按文档 Day 2 拿到供应商实拍图后，在后台产品编辑里替换图片路径（图片放 `public/img/products/`），或直接新建产品。
7. **像素**：后台"设置→数据追踪"填 GA4/Meta/TikTok ID → 下一笔测试单确认 Purchase 事件。
8. **过一遍后台"上线验收"13 项** → 全绿再投广告。

## 技术说明

- Node.js + Express，JSON 文件存储（`data/`），无数据库依赖；订单量上来后再迁 SQLite/Postgres
- 管理密码 scrypt 加盐哈希；订单确认/评价链接 HMAC 签名防遍历
- 评价系统只接受**真实订单邮件邀请链接**提交 + 后台审核，从机制上杜绝假评（文档 §11）
- `npm run images` 重新生成演示插画

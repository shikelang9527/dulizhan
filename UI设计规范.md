# MewMew 宠物独立站 UI 设计规范

版本：v1.0  
适用范围：前台商城、中文/英文页面、后台管理系统、GitHub Pages 静态展示版  
核心目标：温暖、可信、精致、好操作，让用户觉得这是一个认真经营的宠物品牌，而不是模板站。

## 1. 设计定位

### 品牌气质

MewMew 的视觉应该像“专业但不冷冰冰的宠物生活方式品牌”。

- 前台：温暖、亲和、干净、轻高级，突出产品、承诺、购物安全感。
- 后台：克制、高效、清楚，突出订单、产品、支付、消息等运营任务。
- 文案：直接、诚实、不夸大功效，不使用医疗、治疗、保证类表达。
- 交互：轻微、有反馈、有层次，不做夸张动画，不影响购买效率。

### 用户感受关键词

- 可信：支付、退货、物流、客服说明要明确。
- 舒适：颜色温暖，页面留白充足，文字易读。
- 高级：动效轻、层次清楚、卡片阴影克制、按钮反馈顺滑。
- 好买：商品、价格、加入购物车、结账入口要始终清楚。

## 2. 视觉基础

### 前台颜色

当前前台主色来自 `public/css/store.css`。

| 用途 | 变量 | 色值 | 使用场景 |
|---|---|---:|---|
| 页面背景 | `--bg` | `#faf6f0` | 全站背景、浅色区块 |
| 卡片/表单底色 | `--surface` | `#ffffff` | 商品卡、表单、购物车、政策页 |
| 主文字 | `--ink` | `#3d3129` | 标题、正文重点、导航 |
| 辅助文字 | `--ink-soft` | `#8c7d6f` | 描述、脚注、次级信息 |
| 分割线 | `--line` | `#f0e6da` | 卡片边框、表格线、输入框 |
| 品牌橙 | `--brand` | `#e8734a` | 主按钮、购物车数量、重点状态 |
| 品牌深橙 | `--brand-dark` | `#d05e36` | 链接、按钮 hover |
| 品牌浅底 | `--brand-soft` | `#fdf1ea` | 商品图底色、支付提示 |
| 辅助绿 | `--accent` | `#7fb69d` | 成功、信任、进度条 |
| 错误红 | `--danger` | `#c0392b` | 表单错误、删除、失败状态 |

使用原则：

- 一个页面最多同时出现 1 个主强调色和 1 个辅助色。
- 大面积背景用浅暖色，主按钮用品牌橙。
- 红色只用于危险、错误、删除，不用于普通提醒。
- 绿色只用于完成、成功、安心承诺，不用于普通装饰。

### 后台颜色

后台基于 `public/css/admin.css` 的 premium layer。

| 用途 | 变量 | 色值 | 使用场景 |
|---|---|---:|---|
| 页面背景 | `--bg` | `#f7f2eb` | 后台整体背景 |
| 面板底色 | `--surface` | `rgba(255,252,247,.88)` | 仪表盘、表格、表单 |
| 主文字 | `--ink` | `#27211b` | 标题、数据、主要内容 |
| 次级文字 | `--ink-soft` | `#817568` | 标签、说明、弱提示 |
| 品牌棕橙 | `--brand` | `#a7683f` | 后台主操作、当前导航 |
| 深棕 | `--brand-dark` | `#6f4429` | hover、链接、强调 |
| 蓝色 | `--blue` | `#416d9f` | 信息、待处理 |
| 绿色 | `--green` | `#377b56` | 成功、完成 |
| 红色 | `--red` | `#b64d3d` | 风险、失败、删除 |
| 橙色 | `--orange` | `#b97735` | 警告、待配置 |

后台使用原则：

- 后台不做“营销页感”，重点是运营效率。
- 数据面板、订单列表、产品表格要清晰可扫读。
- 当前任务要有明显入口，例如“待发货”“评价审核”“客服消息”。
- 危险操作必须使用红色或二次确认。

## 3. 字体与排版

### 字体

前台：

```css
"Segoe UI", -apple-system, BlinkMacSystemFont, "Helvetica Neue", Arial, sans-serif
```

后台：

```css
"Inter", "Segoe UI", "Microsoft YaHei", -apple-system, BlinkMacSystemFont, sans-serif
```

后台品牌标题可使用：

```css
Georgia, "Times New Roman", "Songti SC", serif
```

### 字号建议

| 场景 | 桌面端 | 移动端 | 说明 |
|---|---:|---:|---|
| 首页主标题 | 44-52px | 32-38px | 只用于首页首屏 |
| 页面标题 | 26-30px | 24-28px | 商品列表、政策页、后台页面 |
| 区块标题 | 22-26px | 20-24px | 分类、热卖、承诺 |
| 卡片标题 | 15-17px | 15-16px | 商品卡、后台动作卡 |
| 正文 | 14.5-16px | 14-15px | 商品介绍、政策说明 |
| 辅助文字 | 12-13.5px | 12-13px | 标签、时间、脚注 |

排版规则：

- 正文字高保持 `1.6`，不要堆太密。
- 标题不要使用过度负字距，当前规范要求 `letter-spacing: 0`。
- 商品名、按钮文案、表格字段不要换成花哨字体。
- 中文后台优先使用简体中文，避免中英混杂。

## 4. 布局规范

### 页面宽度

- 前台主要内容宽度：`1120px`
- 窄内容页宽度：`860px`
- 后台主内容最大宽度：`1220px`
- 移动端左右内边距：`16-20px`

### 栅格

前台：

- 商品列表桌面端：4 列
- 平板：2 列
- 手机：1-2 列，优先保证图片和价格清晰
- 分类入口桌面端：5 列
- 信任条桌面端：4 列

后台：

- 后台整体：左侧导航 + 右侧主内容
- 指标卡：桌面 3-4 列，移动端 1 列
- 表单行：优先使用自适应网格，最小列宽 `120px`
- 表格：信息密度适中，行 hover 只做浅色提示

### 留白

- 页面区块上下间距：`44px` 左右。
- 卡片内边距：前台 `14-22px`，后台 `16-22px`。
- 表单字段间距：`10-12px`。
- 按钮组间距：`8-12px`。

## 5. 组件规范

### 按钮

按钮分三类：

| 类型 | 类名 | 用途 |
|---|---|---|
| 主按钮 | `.btn-primary` | 加入购物车、结账、保存、主要提交 |
| 次按钮 | `.btn-ghost` | 返回、筛选、辅助操作 |
| 危险按钮 | `.btn-danger` | 删除、退款、危险操作 |

前台按钮：

- 圆角使用胶囊形，适合轻松的商城气质。
- 主按钮带轻阴影，hover 上移 `1px`。
- 禁用状态必须变灰，并取消阴影。

后台按钮：

- 圆角保持 `8-10px`，更偏工具感。
- 主按钮不宜过大，强调效率。
- 删除、退款必须使用危险按钮，并加确认。

### 商品卡

商品卡必须包含：

- 商品图
- 商品名
- 分类或标签
- 价格
- 对比价，如有
- 快速加入购物车按钮

交互要求：

- hover 时轻微上移，不超过 `3px`。
- 阴影要浅，不做厚重发光。
- 图片比例固定为 `1:1`，避免列表跳动。
- 加购按钮要有即时反馈，如 toast 或购物车抽屉更新。

### 表单

表单规范：

- label 必须清楚说明字段含义。
- 必填字段应在 HTML 上使用 `required`。
- 输入框 focus 使用品牌色边框。
- 错误提示使用红色浅底 `.form-error`。
- 成功提示使用绿色浅底 `.form-success`。

后台表单要求：

- 保存按钮固定在表单末尾。
- 分组标题要明确，例如“支付设置”“物流设置”“SMTP 设置”。
- 支付、退款、密码等高风险操作必须给出明确反馈。

### 表格

后台表格用于订单、产品、评价、消息、邮件、广告数据。

规范：

- 表头使用弱色文字。
- 行 hover 使用浅暖底。
- 状态使用 `.st` 标签，不只依赖文字。
- 金额、库存、状态、操作入口必须容易扫读。
- 表格中不要放太长说明，详情进入详情页。

### 标签与状态

状态色语义：

- 灰色：草稿、未启用、普通状态
- 蓝色：进行中、待处理
- 绿色：成功、已完成、已上架
- 橙色：警告、待配置、需要关注
- 红色：失败、风险、删除、退款风险

所有状态必须有文字，不允许只用颜色表达。

## 6. 高级交互感规范

高级感来自“反馈准确 + 动效克制 + 层次稳定”，不是动画越多越好。

### 必须保留的交互

| 交互 | 位置 | 目的 |
|---|---|---|
| 商品卡渐入 | 商品列表 | 让页面加载更柔和 |
| 商品卡 hover 上移 | 商品卡 | 提示可点击 |
| 加购 toast | 加入购物车 | 告诉用户操作成功 |
| 购物车抽屉 | 全站 | 不离开当前页面即可确认购物车 |
| 免邮进度条 | 购物车抽屉/购物车页 | 提升购买决策 |
| 数量步进器 | PDP/购物车 | 降低输入成本 |
| 表单提交 loading | 后台表单 | 防止重复提交 |
| 后台导航 active | 后台 | 明确当前位置 |
| 后台动作卡 | 仪表盘 | 把运营任务变成入口 |

### 动效时间

| 动效 | 时长 |
|---|---:|
| hover 反馈 | `120-180ms` |
| 卡片渐入 | `220-360ms` |
| 抽屉打开 | `240-320ms` |
| toast 停留 | `2200-2800ms` |
| 进度条变化 | `250-350ms` |

### 动效限制

- 不做整页大幅移动。
- 不做影响文字阅读的闪烁。
- 不做循环吸引注意力动画，除非是加载中。
- 移动端减少复杂 hover 依赖。
- 交互反馈必须帮助用户理解结果。

## 7. 前台页面规范

### 首页

首页首屏必须包含：

- 品牌识别：MewMew
- 核心价值：精选、诚实描述、30 天退货
- 明确 CTA：逛商品、品牌故事
- 视觉资产：宠物/产品相关图像
- 信任条：免邮、退货、物流、客服

首页内容顺序建议：

1. 顶部公告
2. 导航
3. Hero
4. 信任条
5. 分类入口
6. 热卖/新品
7. 品牌承诺
8. 页脚订阅与政策入口

### 商品列表页

必须保证：

- 分类筛选清楚
- 商品图统一比例
- 价格明显
- 加购入口易点
- 移动端不拥挤

### 商品详情页

必须包含：

- 商品大图和缩略图
- 标题、价格、库存/状态
- 一句话卖点
- 适合谁
- 具体利益点
- 配送和退货说明
- 规格表
- FAQ
- 真实评价机制说明

### 购物车与结账页

必须包含：

- 商品明细
- 数量修改
- 删除
- 小计/运费/总计
- 免邮进度
- 支付方式说明
- 静态演示版需要提示“结账需要服务器版”

## 8. 后台页面规范

### 后台整体

后台语言必须中文为主。

导航结构：

- 仪表盘
- 订单
- 产品
- 优惠码
- 评价审核
- 订阅者
- 客服消息
- 邮件中心
- 广告测试
- 上线验收
- 设置

### 仪表盘

仪表盘不是展示页，必须能直接做事。

必须包含：

- 今日/近期核心指标
- 待处理任务
- 快捷动作卡
- 最近订单
- 产品健康状态
- 上线验收进度

重点任务优先级：

1. 风控订单
2. 待发货订单
3. 客服消息
4. 评价审核
5. 支付/SMTP/像素配置

### 订单页

订单页必须能完成：

- 按状态筛选
- 查看订单详情
- 标记发货
- 标记送达
- 退款
- 记录备注
- 查看风控提示

### 产品页

产品页必须能完成：

- 新建产品
- 编辑价格/库存/状态
- 编辑中文文案
- 查看选品评分
- 草稿/上架管理
- 删除前确认

### 设置页

设置页必须分组：

- 店铺信息
- 物流设置
- 支付设置
- 数据追踪像素
- SMTP 邮箱
- 风控阈值
- 管理员密码

支付设置必须支持：

- 测试支付
- Stripe
- PayPal
- 支付宝国际收款，需 Stripe 支持

## 9. 响应式规范

### 桌面端

- 导航横向展示。
- 商品列表 4 列。
- PDP 双栏布局。
- 结账页双栏，订单摘要 sticky。
- 后台左侧导航固定。

### 平板端

- 商品列表 2 列。
- PDP 改为单栏。
- 结账页订单摘要置顶或单栏。
- 后台导航可折行。

### 手机端

- 导航折叠。
- 商品卡减少信息密度，但保留价格和加购。
- 按钮宽度要适合拇指点击。
- 表格必要时改为卡片式，避免横向滚动过多。
- 弹窗和抽屉不能遮住主要操作。

## 10. 可访问性规范

必须做到：

- 图片有 `alt`，商品图 alt 使用商品名。
- 图标按钮必须有 `aria-label`。
- 抽屉使用 `aria-hidden` 管理状态。
- 表单字段有 label。
- 颜色不能作为唯一状态表达。
- 按钮和链接键盘可访问。
- 文字和背景对比度足够。
- 错误信息要靠近对应操作区域。

## 11. 文案规范

### 前台文案

风格：

- 简洁
- 诚实
- 有温度
- 不夸大

避免：

- “100% 治愈”
- “永久有效”
- “保证到达”
- “全网最低”
- “医学级效果”

推荐表达：

- “预计 7-15 个工作日”
- “30 天未使用可退”
- “真人客服 1 个工作日内回复”
- “我们不会看到或保存你的银行卡号”

### 后台文案

后台文案要动作明确：

- “保存”
- “标记发货”
- “标记已送达”
- “退款”
- “通过”
- “拒绝”
- “标记已处理”

不要使用模糊文案：

- “确定”
- “处理一下”
- “操作”
- “提交这个”

## 12. 图片与素材规范

当前站点使用 SVG 插画作为演示图。

正式运营建议：

- 商品主图换成真实产品图。
- 图片比例统一为 `1:1`。
- 每个商品至少 4 张图：主图、细节图、场景图、尺寸/使用图。
- 不使用低清、水印、供应商拼接图作为首图。
- 首图背景尽量干净，产品主体清楚。
- 宠物图不要遮挡产品本身。

## 13. 设计验收清单

每次新增页面或改版前，按下面检查：

- 页面首屏是否能看出品牌和当前任务？
- 主按钮是否只有一个最重要动作？
- 商品价格、库存、加购是否清楚？
- 移动端按钮和文字是否不拥挤？
- 表单错误是否清楚？
- 加购、保存、删除、退款是否有反馈？
- 图片比例是否统一？
- 后台页面是否中文化？
- 状态是否有文字和颜色双重表达？
- GitHub Pages 静态版是否没有后台/真实支付误导？

## 14. 当前需要继续优化的点

按优先级建议：

1. 前台正式运营前替换真实产品图，当前 SVG 更适合演示。
2. 后台可以继续减少 emoji 的使用，让工具感更稳定。
3. 商品列表移动端可进一步优化卡片高度，避免标题长短导致视觉不齐。
4. GitHub Pages 静态版需要始终保留“演示站”提示，避免用户误以为能真实下单。
5. 后续如果接真实支付，结账页需要补充支付失败、支付取消、支付成功回跳状态页。
